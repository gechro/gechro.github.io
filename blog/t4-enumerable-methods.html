Enumerable#map

Find a Way!

March 1, 2015

Hey, welcome back!

So last week we got an intro to Ruby and learned about arrays and hashes. This week I took a look at the Array and Hash classes, and became more familiar with some of the methods in those classes. While exploring Arrays and Hashes, I came across the Enumerable class.

Enumerable is a module in Ruby that contains a list of many more methods. These methods can be used by classes that hold their own #each method. The #each method should be able to pass each item in the collection, through a code block. Because the Array and Hash classes have their own each methods, we can use enumerable methods. One of the coolest enumerable methods I came across this past week was #map.

The Enumerable#map method returns a new array with the results of running block once for every element in enum (according to Ruby-Doc). What does this mean? It means that map takes each object listed in an array or hash, or any other class that takes #map, and it applies the code block to each item. Let’s look at an example of calling map on an array:

group_4 = [“sherman”, “sid”, “tanya”, “will”, “geovanna”]

new_array = group_4.map {|name| name.capitalize}

Looks like someone lazily added names to group_4, didn’t even bother to capitalize. So, the code above, will take the objects in our array, group_4, go through each one and and return a new array with the names capitalized. Check it out:

group_4 = [“sherman”, “sid”, “tanya”, “will”, “geovanna”]

New_array = [“Sherman”, “Sid”, “Tanya”, “Will”, “Geovanna”]

So, we see that the original array remained the same. Our new array is what has changed. #map works on hashes too. We just have to make sure we specify what we want to run the code on:

group_hash = {
“person1”=>“sherman”,
“person2”=>“sid”,
“person3”=>“tanya”,
“person4”=>“will”,
“person5”=>“geovanna”
}

group_array = group_hash.map {|role,name| [role.upcase, name.capitalize]}

So again, this code is calling for the names to be capitalized, but it wants the role to be all caps. Because this is a hash, it should return a new array, with nested arrays consisting of the key/value pairs, with the code block applied. The original hash remains untouched:

group_hash = {
“person1”=>“sherman”,
“person2”=>“sid”,
“person3”=>“tanya”,
“person4”=>“will”,
“person5”=>“geovanna”
}

group_array= [[“PERSON1”,“Sherman”] [“PERSON2”,“Sid”],[“PERSON3”,“Tanya”], [“PERSON4”,“Will”],[“PERSON5”, “Geovanna”]]

And the Enumerable#map is that simple! The possibilities are endless with this one method. Imagine what you can do with all of the other methods out there! What do you think of the map method? Ready to try it out?
